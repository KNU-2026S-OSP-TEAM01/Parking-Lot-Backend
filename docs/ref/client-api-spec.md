# 클라이언트 팀에게 - 서버 전송 API 명세

> 작성일: 2026-05-03  
> 대상: 주차장 카메라 클라이언트 담당

---

## 개요

카메라가 번호판을 인식하면 **즉시** Parking Lot Server로 HTTP 요청을 전송합니다.  
**입차/출차 판단은 서버가 합니다.** 클라이언트는 번호판과 인식 시각만 보내면 됩니다.

---

## 번호판 전송 API

### 요청

```
POST /api/v1/plates
```

**헤더**

| 헤더 | 값 | 설명 |
|------|----|------|
| `Content-Type` | `application/json` | |
| `Authorization` | `Bearer {api_key}` | 주차장별 발급 키. 서버 관리자에게 받아야 함 |

**바디**

```json
{
  "plate": "12가3456",
  "timestamp": "2026-05-03T10:00:00+09:00"
}
```

| 필드 | 타입 | 설명 |
|------|------|------|
| `plate` | `string` | 인식된 번호판 문자열. 공백 제거 후 전송 |
| `timestamp` | `string` | 카메라가 인식한 시각. **ISO 8601, 타임존 포함** 필수 |

---

### 응답

**입차로 처리된 경우 — `200 OK`**

```json
{
  "event": "entry",
  "entered_at": "2026-05-03T10:00:00+09:00"
}
```

**출차로 처리된 경우 — `200 OK`**

```json
{
  "event": "exit",
  "fee": 3000,
  "parked_duration_minutes": 90
}
```

**인증 실패 — `401 Unauthorized`**

```json
{
  "error": "invalid_api_key"
}
```

**잘못된 요청 — `400 Bad Request`**

```json
{
  "error": "invalid_request",
  "message": "plate is required"
}
```

---

## 주의사항

1. **타임존 누락 금지** — `timestamp`는 반드시 `+09:00` 또는 `Z` 포함. 누락 시 400 반환.
2. **재전송 금지** — 응답을 못 받았다고 동일 번호판을 재전송하면 입출차가 반전될 수 있음. 재전송 전 반드시 서버 상태 확인 필요 (추후 멱등성 API 검토 예정).
3. **api_key 관리** — api_key는 주차장별로 다름. 하드코딩하지 말고 설정 파일로 관리.
4. **번호판 형식** — 공백 없이 전송. 예: `"12가3456"` (O), `"12 가 3456"` (X).
