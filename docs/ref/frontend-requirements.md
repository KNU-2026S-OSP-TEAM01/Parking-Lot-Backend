# 프론트엔드 팀에게 - 요구사항 정의 요청

> 작성일: 2026-05-03

---

## 배경

백엔드는 두 개의 서버로 구성됩니다.

- **Parking Lot Server** — 개별 주차장 입출차 관리 (관리자 전용)
- **Hub Server** — 여러 주차장 현황을 통합해 외부 사용자에게 제공

프론트엔드 팀은 아래 두 영역에서 화면을 담당합니다.

---

## 요청 1. 주차장 현황 페이지 (Hub 기반 - 일반 사용자용)

### 제공 가능한 데이터

Hub Server는 등록된 주차장별로 다음 데이터를 제공할 수 있습니다.

| 데이터 | 설명 |
|--------|------|
| 주차장 이름 | - |
| 주소 | - |
| 위도/경도 | 지도 연동 가능 |
| 전화번호 | - |
| 운영 시간 | - |
| 기본 요금 | 원 단위 정수 |
| 기본 요금 적용 시간 | 분 단위 정수 |
| 단위 추가 요금 | 원 단위 정수 |
| 추가 요금 단위 시간 | 분 단위 정수 |
| 일일 최대 요금 | 원 단위 정수. `null`이면 상한 없음 |
| 전체 면수 | - |
| 잔여 면수 | 폴링 또는 웹훅으로 실시간 갱신 |
| 마지막 동기화 시각 | 데이터 신선도 표시용 |

### 요청 사항

1. **화면 디자인(와이어프레임 또는 목업)** 을 공유해 주세요.  
   - 목록 화면과 상세 화면 각각 필요한지 확인 부탁드립니다.
2. **필요한 API 엔드포인트 목록**을 정리해 주세요.  
   예시: `GET /lots`, `GET /lots/{id}`, 필터링/정렬 파라미터 등
3. 실시간 잔여 면수 표시 방식을 결정해 주세요.  
   - **폴링** (주기적 요청) vs **SSE/웹소켓** (서버 푸시) — 프론트 구현 난이도에 맞춰 선택해 주세요.

---

## 요청 2. 관리자 페이지 (Parking Lot Server 기반)

관리자는 다음 정보를 조회·관리해야 합니다.

- 현재 주차 중인 차량 목록 (번호판 복호화해서 표시)
- 입출차 로그 조회 (날짜 필터, 번호판 검색)
- 주차장 기본 정보 수정

### 요청 사항

1. 각 페이지의 **화면 디자인**을 공유해 주세요.
2. 필요한 **API 엔드포인트 목록과 응답 데이터 구조** 요구사항을 정리해 주세요.
3. 관리자 로그인 화면 디자인도 포함해 주세요.

---

## 전달 형식

와이어프레임 + 필요 API 목록을 **Figma, Notion, 또는 마크다운** 중 편한 형식으로 공유해 주세요.
